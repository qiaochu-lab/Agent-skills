# CSO Recon Standards

侦察级别（recon_level）和置信度（Confidence Grading）标准定义。

---

## [GB-01] Recon Level（侦察级别）

侦察级别定义了情报收集的信息源边界。

### 完整定义

**L1 (基础 - Basic)**: 仅限官网/公告

**信息源范围**:
- 官方网站
- 政府公告
- 上市公司财报
- 官方新闻稿

**特点**:
- ✅ 权威性高
- ✅ 可靠性强
- ❌ 信息滞后
- ❌ 覆盖范围窄

**适用场景**:
- 财报分析
- 政策研究
- 合规调研
- 官方声明验证

**L2 (关联 - Associated)**: L1 + 主流财经媒体

**信息源范围**:
- L1 的所有来源
- 主流财经媒体（Bloomberg、Reuters、WSJ、FT）
- 行业权威媒体（TechCrunch、The Verge）
- 专业研究机构报告（Gartner、IDC）

**特点**:
- ✅ 平衡权威性与时效性
- ✅ 覆盖面较广
- ⚠️ 可能有媒体偏见
- ⚠️ 需要交叉验证

**适用场景**:
- 市场研究
- 行业分析
- 竞争情报
- 趋势识别

**L3 (全网 - Full Coverage)**: L2 + 社交媒体/传闻

**信息源范围**:
- L2 的所有来源
- 社交媒体（Twitter/X、LinkedIn、Reddit）
- 行业论坛
- 传闻和内部消息
- 非官方渠道

**特点**:
- ✅ 覆盖全面
- ✅ 时效性最强
- ❌ 噪音多
- ❌ 必须严格验证

**适用场景**:
- 舆情监测
- 趋势预判
- 突发事件追踪
- 新兴话题发现

## [GB-02] Confidence Grading（置信度分级）

置信度定义了情报的可靠性等级。

### 完整定义

**S级（Supreme - 最高）**: 回测证实的档案规律

**定义**: 通过历史数据回测验证的规律性模式

**数据源要求**:
- 历史数据库（至少 3 年）
- 量化验证（统计显著性 p < 0.05）
- 多周期回测

**使用场景**:
- 量化交易策略
- 风险模型
- 预测模型验证

**置信度**: 最高（经过验证）

**风险提示**: 无（但需注意过拟合风险）

---

**A级（Authoritative - 权威）**: 24H 内权威实时数据

**定义**: 24 小时内发布的权威机构数据

**数据源要求**:
- 官方机构（政府、央行、交易所）
- 上市公司公告
- 权威研究机构
- 时效性 ≤ 24 小时

**使用场景**:
- 交易决策
- 突发事件分析
- 实时监控
- 政策影响评估

**置信度**: 高（权威源 + 时效性）

**风险提示**: 无

---

**B级（Balanced - 均衡）**: 样本充足的档案证据或当前情报

**定义**: 多源交叉验证的档案数据或当前情报

**数据源要求**:
- 至少 3 个独立信源
- 主流媒体报道
- 行业分析报告
- 历史数据支持

**使用场景**:
- 趋势分析
- 中长期规划
- 市场研究
- 战略决策

**置信度**: 中等（需交叉验证）

**风险提示**: 建议交叉验证多个来源

---

**C级（Caution - 谨慎）**: 过期的实时数据（>7天）

**定义**: 发布时间超过 7 天的数据或单一信源情报

**数据源要求**:
- 时效性 > 7 天的权威源
- 或单一信源的当前情报
- 或非权威媒体报道

**使用场景**:
- 背景研究
- 历史分析
- 探索性研究
- 趋势识别（非决策依据）

**置信度**: 较低（时效性降低）

**风险提示**: **必须标注"需强风险提示"**

---

**D级（Dangerous - 危险）**: 逻辑毒素（矛盾数据/AI 脑补数据）

**定义**: 自相矛盾或未经验证的推测性数据

**数据源**:
- 互相矛盾的信息
- AI 生成的未验证内容
- 无法溯源的数据
- 明显虚假信息

**使用场景**: **禁止使用**

**置信度**: 无

**风险提示**: **触发物理熔断，严禁载入计算层**

## 选择指南

### 根据任务类型选择 recon_level

| 任务类型 | 推荐 Level | 理由 |
|---------|-----------|------|
| 财报分析 | L1 | 需要官方数据，不依赖媒体解读 |
| 政策研究 | L1 | 需要权威官方信息 |
| 市场研究 | L2 | 需要媒体报道补充，了解市场反应 |
| 行业分析 | L2 | 需要多方观点，平衡官方和媒体 |
| 竞争情报 | L2-L3 | 需要全面信息，包括非官方渠道 |
| 舆情监测 | L3 | 需要社交媒体，捕捉公众情绪 |
| 趋势预判 | L3 | 需要全网信息，发现新兴趋势 |
| 突发事件 | L3 | 需要实时信息，社交媒体最快 |

### 根据任务关键性选择置信度门槛

| 任务关键性 | 最低门槛 | 理由 |
|-----------|---------|------|
| 关键决策（投资、政策） | A 级 | 需要最高可靠性 |
| 战略规划（市场研究） | B 级 | 需要多源验证 |
| 背景分析（行业概览） | C 级 | 可接受较低时效性 |
| 探索研究（趋势预判） | C 级 | 需标注风险 |
| 量化策略 | S 级 | 需要回测验证 |

## 实际应用示例

### 示例 1: 财报分析任务

```
-recon_level: L1
-置信度门槛: A级

理由:
- L1: 只需要官方财报数据
- A级: 财报发布后 24H 内，数据权威且实时
```

### 示例 2: 市场研究任务

```
-recon_level: L2
-置信度门槛: B级

理由:
- L2: 需要官方数据 + 媒体报道，了解市场反应
- B级: 多源交叉验证，平衡可靠性和覆盖面
```

### 示例 3: 舆情监测任务

```
-recon_level: L3
-置信度门槛: C级

理由:
- L3: 需要社交媒体数据，捕捉公众情绪
- C级: 社交媒体信息时效性强但需验证，标注风险
```

### 示例 4: 量化策略开发

```
-recon_level: L1
-置信度门槛: S级

理由:
- L1: 需要官方历史数据（交易所、财报）
- S级: 必须通过回测验证，不接受未验证的规律
```

## 数据降级规则

**时间降级**:
- A 级数据在 7 天后自动降级为 C 级
- C 级数据在 30 天后应重新评估是否降级为 D 级

**信源降级**:
- 单一信源的 B 级数据应降级为 C 级
- 无法验证的 C 级数据应降级为 D 级

**矛盾降级**:
- 发现矛盾信息时，所有相关数据立即降级为 D 级
- 触发逻辑熔断，停止计算

## 引用格式

在 P-Prompt 中引用这些标准时：

```
-Target_Data: "[为逻辑闭环所需的情报]"
--1、全球AI芯片市场份额变化 | 时间窗: [20250101-20260118] | 置信度门槛: A级
--2、供应链动态 | 时间窗: [20250101-20260630] | 置信度门槛: B级
-recon_level: L2
```
